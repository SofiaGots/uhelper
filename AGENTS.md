# Спецификация AI-агентов для Ассистента по поступлению в университет

## Обзор проекта
**Проект:** Ассистент для персонализированной поддержки на всех этапах поступления в университет
**Тип:** Telegram-бот (MVP) → Веб-платформа (будущее)
**Целевые пользователи:** Ученики 9-11 классов, готовящиеся к поступлению в вузы (Россия и за рубежом)
**Сроки реализации:** Сентябрь 2025 - Февраль 2026

- Прямые коммиты в `main` запрещены

#### Процесс внесения изменений
1. **Создание ветки:**
   ```bash
   git checkout -b feature/название-фичи
   ```

2. **Внесение изменений и коммит:**
   ```bash
   git add .
   git commit -m "Описание изменений"
   ```

3. **Отправка ветки на GitHub:**
   ```bash
   git push -u origin feature/название-фичи
   ```

4. **Создание Pull Request:**

## Архитектура основных агентов

### 1. Главный оркестрационный агент (`OrchestratorAgent`)
**Цель:** Центральный координатор, маршрутизирующий запросы пользователя к соответствующим специализированным агентам

**Обязанности:**
- Распознавание и классификация намерений пользователя
- Управление сессиями и отслеживание контекста
- Выбор и маршрутизация агентов
- Агрегация и форматирование ответов
- Обработка ошибок и резервные механизмы

**Вход/Выход:**
- Вход: Сырые сообщения пользователя из Telegram
- Выход: Структурированные ответы с соответствующей маршрутизацией агентов

**Ключевые возможности:**
- Понимание естественного языка для обнаружения намерений
- Контекстно-зависимая маршрутизация на основе истории разговора
- Координация нескольких агентов
- Управление состоянием пользователя

---

### 2. Агент информации о университетах (`UniversityDataAgent`)
**Цель:** Предоставляет комплексную информацию о университетах и программах

**Обязанности:**
- Запросы к базе данных информации о университетах
- Поиск программ и требований к поступлению
- Информация о стипендиях и финансировании
- Данные о международных университетах (Россия и за рубежом)

**Источники данных:**
- Базы данных и API университетов
- Государственные образовательные порталы
- Базы данных стипендий
- Обновления о поступлении в реальном времени

**Ключевые возможности:**
- Поиск и фильтрация университетов
- Сравнение программ
- Отслеживание сроков поступления
- Оценка соответствия критериям стипендий

**Примеры обрабатываемых запросов:**
- "Какие университеты в Москве с сильной компьютерной наукой?"
- "Какие требования для поступления в МГУ на факультет журналистики?"
- "Стипендии для иностранных студентов в Германии"

---

### 3. Агент анализа личного профиля (`ProfileAnalyzerAgent`)
**Цель:** Анализирует профили студентов и предоставляет персонализированные рекомендации

**Обязанности:**
- Анализ академической успеваемости
- Оценка интересов и способностей
- Оценка навыков и внеучебной деятельности
- Персонализированный подбор университетов

**Обработка данных:**
- Анализ оценок и расчет GPA
- Идентификация сильных предметов
- Оценка влияния внеучебной деятельности
- Сопоставление карьерных интересов

**Выходные возможности:**
- Оценки соответствия университетам
- Прогнозы вероятности успеха
- Анализ сильных и слабых сторон
- Персонализированные рекомендации по улучшению

**Собираемые данные пользователя:**
- Академические записи (оценки, предметы)
- Результаты тестов (ЕГЭ, SAT и др.)
- Интересы и карьерные предпочтения
- Внеучебная деятельность
- Географические предпочтения

---

### 4. Агент сроков поступления (`TimelineAgent`)
**Цель:** Управляет сроками поступления и создает персонализированные графики

**Обязанности:**
- Отслеживание сроков и уведомлений
- Создание персонализированных графиков
- Мониторинг прогресса
- Генерация напоминаний

**Ключевые возможности:**
- Агрегация сроков для нескольких университетов
- Планирование на основе приоритетов
- Автоматические напоминания (ежедневные/еженедельные)
- Визуализация прогресса

**Компоненты графика:**
- Сроки подачи заявлений
- Графики подготовки к экзаменам
- Временные рамки подачи документов
- Периоды подачи заявок на стипендии
- Сроки обработки виз (для международных студентов)

---

### 5. Агент подготовки к экзаменам (`ExamPrepAgent`)
**Цель:** Помогает с подготовкой к экзаменам и разработкой стратегии

**Обязанности:**
- Руководство по подготовке к ЕГЭ/SAT/другим экзаменам
- Создание планов обучения
- Генерация практических вопросов
- Отслеживание результатов

**Поддерживаемые экзамены:**
- Российские ЕГЭ (различные предметы)
- Международные экзамены (SAT, ACT, IELTS, TOEFL)
- Вступительные экзамены конкретных университетов

**Возможности:**
- Персонализированные графики обучения
- Генерация практических тестов
- Идентификация слабых областей
- Отслеживание прогресса и аналитика

---

### 6. Агент управления документами (`DocumentAgent`)
**Цель:** Управляет документами заявлений и требованиями

**Обязанности:**
- Создание контрольных списков документов
- Генерация шаблонов (эссе, резюме, мотивационные письма)
- Проверка документов и обратная связь
- Проверка форматов

**Типы документов:**
- Формы заявлений
- Эссе и личные заявления
- Рекомендательные письма
- Транскрипты и сертификаты
- Материалы портфолио

**Возможности:**
- Настройка шаблонов документов
- Проверка грамматики и стиля
- Форматирование под конкретные университеты
- Планирование документов на основе сроков

---

### 7. Интерактивный агент оценки (`AssessmentAgent`)
**Цель:** Проводит интерактивные оценки и тесты

**Обязанности:**
- Тестирование личности и способностей
- Оценка соответствия университетам
- Исследование карьерных интересов
- Анализ пробелов в навыках

**Типы оценок:**
- Опросники карьерных интересов
- Оценки стилей обучения
- Тесты на совместимость с университетами
- Тесты идентификации сильных сторон

**Интерактивные возможности:**
- Адаптивное тестирование
- Оценка в реальном времени
- Подробные объяснения результатов
- Практические рекомендации

---

### 8. Агент уведомлений и коммуникации (`NotificationAgent`)
**Цель:** Управляет всеми коммуникациями и уведомлениями

**Обязанности:**
- Управление push-уведомлениями
- Интеграция с email/SMS
- Генерация отчетов о прогрессе
- Система оповещений о критических сроках

**Типы уведомлений:**
- Напоминания о сроках (за 7 дней, 3 дня, 1 день)
- Обновления прогресса
- Оповещения о важных объявлениях
- Персонализированные советы и предложения

**Каналы:**
- Сообщения Telegram
- Email-уведомления
- SMS-оповещения (опционально)
- Интеграция с календарем

## Протокол коммуникации агентов

### Формат сообщений
```json
{
  "user_id": "string",
  "session_id": "string",
  "intent": "university_search | profile_analysis | timeline_management | ...",
  "message": "оригинальное сообщение пользователя",
  "context": {
    "previous_intents": [],
    "user_profile": {},
    "conversation_history": []
  },
  "agent_response": {
    "agent": "имя_агента",
    "response": "структурированный ответ",
    "next_steps": [],
    "confidence": 0.95
  }
}
```

### Логика выбора агента
1. **Обнаружение намерения**: Анализ сообщения пользователя на предмет основного намерения
2. **Анализ контекста**: Учет истории разговора и состояния пользователя
3. **Сопоставление агента**: Выбор наиболее подходящего агента на основе намерения и контекста
4. **Генерация ответа**: Агент обрабатывает запрос и генерирует ответ
5. **Оркестрация**: Форматирование и доставка ответа через главного агента

```
Вход пользователя → OrchestratorAgent → [Специализированный агент] → Ответ

    ↑                                           ↓
Уведомления ← NotificationAgent ← Обновления графика
```

- **База данных университетов**: Программы, требования, сроки
- **История разговоров**: Логи чатов, взаимодействия пользователей
- **Данные аналитики**: Паттерны использования, метрики успеха
